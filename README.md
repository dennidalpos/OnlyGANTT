# OnlyGANTT

OnlyGANTT is a Windows-oriented web application for managing project Gantt charts by department. It runs as a Node.js/Express server, serves a bundled React browser UI, and supports optional Windows service and x64 MSI packaging.

It is not a standalone desktop app. The supported product surface is a web UI hosted by the local/server Node process.

## Features

- Interactive Gantt chart with projects, phases, milestones, percentages, alerts and Italian holidays.
- Department-based project files under `Data/reparti/`.
- Multi-user editing locks with persisted lock state under `Data/config/locks.json`.
- Local users, admin login, department password protection and password reset flows.
- Optional LDAP authentication with local fallback support.
- Admin settings import/export with LDAP bind password kept out of the versioned seed config.
- JSON import/export for department projects.
- Local React bundle generated by `esbuild`, served from `artifacts/build/client/`.
- Native Windows service support through a self-contained service host.
- x64 MSI and setup bootstrapper packaging through WiX 3.14.1, with Node.js 24 LTS bundled as an offline prerequisite in the setup EXE.

## Requirements

- Windows.
- PowerShell 7+ (`pwsh`).
- Node.js 20 or newer.
- npm.
- Administrator privileges only for Windows service install/uninstall and MSI lifecycle validation.

## Quick Start

```powershell
npm run bootstrap
npm run doctor
npm start
```

The app starts on `http://localhost:3000` by default. `npm start` runs `scripts/compile.ps1` first, then starts `src/server/server.js`.

## Commands

| Command | Purpose |
| --- | --- |
| `npm run bootstrap` | Install dependencies with `npm ci`. |
| `npm run doctor` | Check Node/npm/dotnet, required files, installed dependencies and packaging inputs. |
| `npm run compile` | Build the browser bundle, native service host and runtime manifest. |
| `npm run build` | Run `doctor` and `compile`. |
| `npm run test` | Run the canonical regression checks. |
| `npm run pack` | Build the x64 MSI and setup EXE under `artifacts/packages/`. |
| `npm run publish` | Copy existing package artifacts to `artifacts/publish/local/`. |
| `npm run clean` | Remove managed artifacts, stale local outputs and runtime-only data files. |

MSI lifecycle validation is available when the environment can install and remove machine-wide packages:

```powershell
npm run pack -- -RunMsiLifecycleValidation
```

Automated environments can also set `ONLYGANTT_RUN_MSI_TESTS=true`. This repository currently has no versioned GitHub Actions workflow.

## Windows Service

The service scripts are consolidated in `scripts/windows/service.ps1` and use native Windows service management:

```powershell
npm run service:install
npm run service:start
npm run service:stop
npm run service:uninstall
```

The manual service path uses the repository or install root as the application directory. The MSI path installs under `C:\Program Files\OnlyGANTT`, sets `ONLYGANTT_DATA_DIR` to the installed `Data` directory and starts `OnlyGanttWeb` automatically when service installation is selected.

## Packaging

`npm run pack` builds:

- `artifacts/packages/msi/OnlyGantt-<version>-x64.msi`
- `artifacts/packages/setup/OnlyGantt-Setup-<version>-x64.exe`

The MSI is x64-only by design: the WiX source uses `ProgramFiles64Folder` and `Platform="x64"`.

The installer:

- installs per machine;
- checks Node.js from HKLM and blocks with precise instructions when using the MSI without Node.js;
- installs Node.js 24 LTS automatically when using the setup EXE;
- asks for service installation, server port, desktop shortcut URL and optional initial admin reset code;
- installs the server, public assets, dependencies, native service host and Windows service configuration;
- creates an all-users desktop shortcut to the configured homepage URL;
- uses the Windows brand icon from `src/public/brand/onlygantt.ico`;
- stores package outputs under `artifacts/packages/`.
